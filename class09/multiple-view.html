Introduction to Multiple Views

In Vue Router, the standard component displays a single component based on the current route. However, sometimes you need to display multiple components simultaneously for the same URL.

When to use multiple views: Dashboard layouts, complex page structures, or when you need different sections of your page to change independently based on the same route.

What are Named Router Views?

Named router views allow you to define multiple outlet points in your template where different components can be rendered simultaneously for the same route.

2. Route Configuration

Use the components property (plural) instead of component (singular):

const routes = [ { path: '/dashboard', components: { default: DashboardLayout, // for unnamed router-view header: DashboardHeader, // for name="header" sidebar: DashboardSidebar, // for name="sidebar" main: DashboardMain, // for name="main" footer: DashboardFooter // for name="footer" } } ]
Important: Use components (plural) in your route configuration when working with named views, not component (singular).

Advanced Usage Patterns

1. Mixing Named and Default Views

You can combine unnamed (default) and named router views in the same template:

{ path: '/profile', components: { default: UserProfile, // renders in unnamed router-view modal: ProfileModal, // renders in name="modal" notifications: UserAlerts // renders in name="notifications" } }

2. Nested Named Views

Named views can also be used with nested routes:

// Parent route with named views { path: '/settings', components: { header: SettingsHeader, main: SettingsLayout }, children: [ { path: 'account', components: { // These will render in SettingsLayout's router-views default: AccountSettings, sidebar: SettingsSidebar } } ] }

3. Dynamic Component Loading

Use dynamic imports for code splitting:

{ path: '/analytics', components: { header: () => import('./components/AnalyticsHeader.vue'), sidebar: () => import('./components/AnalyticsSidebar.vue'), main: () => import('./components/AnalyticsDashboard.vue') } }
